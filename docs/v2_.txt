BIT2

* LOGFILE

  Tartalom:
    ID EVENT DATE CHECKSUM
  
    A teljes könyvelés idősoros naplója. Ebből a teljes index adatbázis
  bármikor előállítható. A napló bejegyzések egyedi ID-t, időbélyeget
  és checksum értéket tartalmaznak, amelyekből bloklánc szerűen
  támaszkodnak egymásra a bejegyzett elemek. 

  - mi lesz a csatolt fileokkal? A tartalmukkal?

* INDEX

    Fájlrendszer mintájára kialakított index file, amely tartalmaz
  minden könyvelési adatot, és a felhasználáshoz szükséges előre
  letárolt állapotot. Gyors, teljes, pontos. A fájl használatához
  részleges olvasási és írási műveleteket hajtunk végre, egy
  fájlrendszer mintájára.

  - tartalmazza a fileokat is? Kezelünk verziókat?

* Egyedülálló / Decentralizált

    GIT szerűen lenne egy központi szerver, amely összehangolná a kliensek
  munkáját. Jóváhagyná a változtatásokat és ahonnan bármikor le lehet
  kérni a teljes könyvelést.

Rugalmas ID

    Szabadszavas, szóköz nélküli azonosítók. Perjel használata lehetséges,
  így akár domain neveket is kezelhetünk.
  
  Pl.:
    invoice/INVOICE_ID
    contract/CONTRACT_ID

Hibrid valóság

    Láthatunk fájlokat fájlrendszer szinten, amiket böngészhetünk, valamint
  ténylegesen dolgozunk egy saját fájlrendszerrel, amelyből az előbbit elő-
  állítjuk.

CLI

    Parancssoros interface, amely platform függetlenül (Windows, Mac, Linux)
  képes kezelni a BIT állományokat, és támogatni a hatékony munkavégzést.
